# [USENIX ATC '25] [SwCC: Software-Programmable and Per-Packet Congestion Control in RDMA Engine](https://www.usenix.org/conference/atc25/presentation/huang-hongjing)

## 四类 RDMA 拥塞控制算法实现

- ASIC-based：
    - **原理**：由 ASIC 网卡实现拥塞控制逻辑，当拥塞通知到达网卡后，网卡根据内置的算法调整发送速率。
    - **优点**：控制逻辑响应快。
    - **缺点**：控制逻辑固定，难以修改和扩展。算法发展速度快，但硬件更新慢，导致算法落后。
- CPU-based：
    - **原理**：由主机 CPU 处理拥塞通知，并调整发送速率。
    - **优点**：控制逻辑灵活，可编程。
    - **缺点**：响应时间较长，严重影响性能。
- FPGA-based：
    - **原理**：使用 FPGA 实现拥塞控制逻辑，能够在硬件层面进行编程。
    - **优点**：兼具 ASIC 的高性能和 CPU 的灵活性。
    - **缺点**：开发复杂度高，成本较高。
- Soc-based：
    - **原理**：将拥塞控制逻辑集成在 SoC（System on Chip）中，结合了 CPU 和硬件加速的优势。
    - **优点**：性能和灵活性兼顾。
    - **缺点**：需要较高的设计水平和调优工作。

这篇论文对 Soc-based 进行了深入研究。
